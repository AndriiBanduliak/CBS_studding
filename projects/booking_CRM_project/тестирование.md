# Чек-лист перед передачей на тестирование

## Подготовка окружения
- Убедиться, что установлен Python 3.12+ и Node.js 18+
- Создать файл `.env` по образцу и заполнить значения: `SECRET_KEY`, `DEBUG`, `ALLOWED_HOSTS`, `GOOGLE_*`
- Применить миграции: `python manage.py migrate`
- Запустить бэкенд: `python manage.py runserver 0.0.0.0:8000`
- Установить зависимости фронтенда: `cd frontend && npm install`
- Запустить фронтенд: `npm run dev -- --host`

## Быстрая проверка здоровья
- Открывается API-документация: `http://127.0.0.1:8000/api/docs/`
- CORS корректно работает с `http://localhost:5173`
- Бэкенд отвечает на `GET /` (или корневой API эндпоинт, если предусмотрен)

## Конфигурация (staging)
- `DEBUG=false`
- `ALLOWED_HOSTS` содержит домен стенда
- `SECRET_KEY` не дефолтный
- Ограничить CORS для staging (не `CORS_ALLOW_ALL_ORIGINS`)
- Заполнены переменные `GOOGLE_*` или интеграции отключены

## Проверка CRUD основных сущностей
- Properties: список, создание, получение, обновление, удаление
- Rates: привязка к объектам, корректность дат и валюты
- Bookings: создание/изменение/отмена, проверка пересечений дат
- Customers: создание, поиск, обновление
- Tasks: статусы, исполнители, дедлайны
- Staff: роли и доступы (если реализованы)
- Notes, Reports, Audit: базовые сценарии чтения/записи

## Валидации и ограничения
- Обязательные поля валидируются, понятные сообщения об ошибках
- Пагинация (PAGE_SIZE=25) работает и управляется параметрами
- Фильтрация/поиск/сортировка через DRF backends функционируют
- Таймзона `Europe/Kyiv`, корректность дат и времени в ответах

## Аутентификация и права
- Вход/выход работает (если авторизация реализована)
- Доступы: анонимный — только чтение, авторизованный — расширенный доступ
- Защита от несанкционированных операций (403/401 где нужно)

## UI фронтенда
- Страницы `Auth`, `Locations`, `Properties` открываются без ошибок в консоли
- Навигация `react-router-dom` работает, ссылки не ломаются

- Адаптивность ключевых экранов (минимальная проверка)
